API-Dokumentation

Base-URL
http://{{ request.get_host }}/api/v1/
Authentifizierung (Header)
Authorization: Bearer <dein-token>
POST /upload/ Dateien hochladen (multipart/form-data, field: "files")
GET /files/ Dateien auflisten (?verein_id=1&kontext=beleg&limit=50)
DELETE /files/<id>/ Datei löschen (permanent!)
GET /info/ Token-Info + Speicher-Statistik

Beispiel: Python

import requests

# 1. Token und URL definieren
TOKEN = "jdcloud_abc123..."
BASE_URL = "http://{{ request.get_host }}/api/v1"

# 2. Header vorbereiten
headers = {"Authorization": f"Bearer {TOKEN}"}

# 3. Datei hochladen
with open("rechnung.pdf", "rb") as f:
    files = {"files": f}
    data = {"verein_id": "1", "kontext": "beleg"}
    response = requests.post(
        f"{BASE_URL}/upload/",
        headers=headers,
        files=files,
        data=data
    )
    print(response.json())  # {"id": 123, "url": "..."}

# 4. Dateien auflisten
response = requests.get(
    f"{BASE_URL}/files/?verein_id=1",
    headers=headers
)
print(response.json())  # [{"id": 123, "name": "rechnung.pdf", ...}]

Beispiel: cURL

# Datei hochladen
curl -X POST "http://{{ request.get_host }}/api/v1/upload/" \
  -H "Authorization: Bearer DEIN_TOKEN" \
  -F "files=@rechnung.pdf" \
  -F "verein_id=1" \
  -F "kontext=beleg"

# Dateien auflisten
curl "http://{{ request.get_host }}/api/v1/files/?verein_id=1" \
  -H "Authorization: Bearer DEIN_TOKEN"
⚠️ Wichtige Hinweise:
  • Tokens sind geheim – behandle sie wie Passwörter!
  • Deaktivierte Tokens funktionieren nicht mehr, können aber reaktiviert werden.
  • Gelöschte Tokens sind unwiderruflich ungültig.
  • Maximale Dateigröße: 100MB pro Datei (kann konfiguriert werden).
  • Der Parameter kontext ist optional und dient der Kategorisierung (z.B. "beleg", "protokoll").
